# Asset Pipeline

This folder is reserved for replaceable authored art.

Current game art is generated by scripts so gameplay can keep moving. Final or intermediate sprite sheets should land here and then replace procedural nodes scene by scene.

## Suggested Layout

- `characters/pizza_fish_taco/`
  - `pizza_fish_taco_idle.png`
  - `source/pizza_fish_taco_idle_source.jpg`
  - `source/pizza_fish_taco_animation_sheet_source.jpg`
  - `frames/*.png`
  - `pizza_fish_taco.png`
  - `pizza_fish_taco.json` or Godot atlas resource
  - animation notes for idle, run, jump, fall, hurt, death, light combo, heavy, and Sauce Spin
- `enemies/burger_grunt/`
  - `source/burger_grunt_sheet_source.png`
  - `frames/*.png`
- `enemies/fry_goblin/`
  - `source/fry_goblin_sheet_source.png`
  - `frames/*.png`
- `bosses/big_bad_burger/`
  - `source/big_bad_burger_sheet_source.png`
  - `frames/*.png`
- `levels/snack_city_streets/`
  - `source/snack_city_panel_0.jpg`
  - `source/snack_city_panel_1.jpg`
  - `source/snack_city_panel_2.jpg`
  - `source/snack_city_panel_3.jpg`
- `ui/`

## Player Animation Contract

`scripts/Player.gd` currently calls these animation names:

- `idle`
- `run`
- `jump`
- `fall`
- `hurt`
- `death`
- `light_1`
- `light_2`
- `light_3`
- `heavy`
- `sauce_spin`

The temporary procedural version lives in `scripts/PlayerArt.gd`. A future `AnimatedSprite2D` or `AnimationPlayer` scene should preserve the same animation names so combat code does not need to change.

## Current Imported Art

`scripts/PlayerArt.gd` now prefers sliced frame PNGs in `characters/pizza_fish_taco/frames/` and falls back to procedural shapes if those files are missing.

Current frame names:

- `idle_0`, `idle_1`, `idle_2`
- `run_0`, `run_1`, `run_2`, `run_3`
- `bite_0`, `bite_1`
- `sauce_0`
- `spin_0`, `spin_1`
- `hurt_0`, `crouch_0`, `jump_0`

To regenerate the older single idle transparent PNG:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\process_player_art.ps1 -Source 'assets\characters\pizza_fish_taco\source\pizza_fish_taco_idle_source.jpg' -Output 'assets\characters\pizza_fish_taco\pizza_fish_taco_idle.png' -TargetHeight 156
```

To regenerate the current sliced animation frames:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\slice_player_sheet.ps1 -Source 'assets\characters\pizza_fish_taco\source\pizza_fish_taco_animation_sheet_source.jpg' -OutputDir 'assets\characters\pizza_fish_taco\frames' -TargetHeight 156
```

## Current Enemy Art

`scripts/Enemy.gd` now prefers sliced frame PNGs for `burger_grunt`, `fry_goblin`, and `big_bad_burger`.

Current Burger Grunt and Fry Goblin frame names:

- `idle_0`
- `walk_0`, `walk_1`
- `windup_0`, `attack_0`
- `hurt_0`, `knockback_0`
- `defeated_0`
- `taunt_0`, `run_0`, `stunned_0`, `recover_0`

To regenerate the sliced Burger Grunt frames:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\slice_enemy_sheet.ps1 -Source 'assets\enemies\burger_grunt\source\burger_grunt_sheet_source.png' -OutputDir 'assets\enemies\burger_grunt\frames' -TargetHeight 96
```

To regenerate the sliced Fry Goblin frames:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\slice_enemy_sheet.ps1 -Source 'assets\enemies\fry_goblin\source\fry_goblin_sheet_source.png' -OutputDir 'assets\enemies\fry_goblin\frames' -TargetHeight 82
```

Current Big Bad Burger frame names:

- `idle_0`
- `walk_0`, `walk_1`
- `chomp_windup_0`, `chomp_attack_0`
- `charge_windup_0`, `charge_attack_0`
- `slam_windup_0`, `slam_impact_0`
- `hurt_0`, `stunned_0`, `defeated_0`

To regenerate the sliced Big Bad Burger frames:

```powershell
powershell -ExecutionPolicy Bypass -Command "& { .\tools\slice_enemy_sheet.ps1 -Source 'assets\bosses\big_bad_burger\source\big_bad_burger_sheet_source.png' -OutputDir 'assets\bosses\big_bad_burger\frames' -FrameNames @('idle_0','walk_0','walk_1','chomp_windup_0','chomp_attack_0','charge_windup_0','charge_attack_0','slam_windup_0','slam_impact_0','hurt_0','stunned_0','defeated_0') -TargetHeight 154 }"
```

## Current Level Art

`scripts/Level.gd` now prefers four illustrated Snack City Streets background plates in `levels/snack_city_streets/source/`. If those files are missing, it falls back to the older procedural background.

The gameplay ground collision is still code-native so visual plate changes do not affect combat or movement.

The current plates are flat full-screen illustrations. The next art pass should tune combat-lane readability, then split the street into parallax background layers and foreground prop sprites.
